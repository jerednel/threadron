Your AI has things to do.

The problem

AI assistants are capable.
Task management hasn't caught up.

You tell Claude to fix a bug. You ask Hermes to book a flight. You have OpenClaw watching your pull requests. But each agent works in isolation. None of them know what the others are doing. And when a conversation ends, the context vanishes.

Tasks for Agents gives your AI a persistent, structured place to track work. Every task carries context that survives across sessions. Every agent has an identity. And you have a board that shows the full picture — across every domain of your life.

How it works

Three ideas. One system.

01

Agents operate

Your AI assistants create tasks, add context, update progress, and mark things done. They pull from a simple REST API that works everywhere — Claude Code, OpenClaw, Hermes, anything.

02

Humans supervise

You see everything on a clean web dashboard. Override priorities. Approve high-stakes actions. Add your own tasks. Your agents work for you — you stay in control.

03

Context persists

Every task carries a structured log of observations, decisions, and artifacts. When an agent picks up a task days later, it has full context. No re-deriving. No lost threads.

Dead simple API

One endpoint. Full picture.

Any agent that can make an HTTP request can use Tasks for Agents. No SDK required.

Create a task POST /v1/tasks { "title": "Deploy staging environment for agentask", "domain": "side-project", "project": "agentask-mvp", "agent": "claude-code", "guardrail": "autonomous", "context": [ { "type": "observation", "body": "Docker compose file ready. Postgres container tested locally." } ] }
Agent checks in GET /v1/tasks?assignee=claude-code&status=pending // Returns tasks assigned to this agent, with full context history. // Agent picks up where it — or another agent — left off.
