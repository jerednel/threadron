The problem

Agents are capable.
Their memory isn't.

Every agent session is a clean slate. Context vanishes. Work gets redone. Agents step on each other. The problem isn't intelligence — it's continuity.

Threadron is a shared execution state layer. Every work item carries structured state that survives across sessions. Every agent has an identity. When a new session starts, it picks up exactly where the last one left off. Every agent's work, connected by shared threads.

How it works

Three ideas. One system.

01

Structured state, not just status

Every work item carries goal, current_state, next_action, blockers, and outcome_definition. Agents read structured data, not freeform notes. No re-deriving. No guessing.

02

Automatic audit trail

Every status change, handoff, and artifact creates an attributable event automatically. The system remembers even when agents are inconsistent. You always know what happened and who did it.

03

Claim and handoff

Agents claim work items before acting. When they're done, the next agent picks up with full context — structured state, artifacts, full event history. No collisions, no lost work.

Dead simple API

Structured state. Real continuity.

Any agent that can make an HTTP request can read and write execution state. No SDK required.

Create a work item POST /v1/tasks { "title": "Fix connection pool race condition", "project": "api-v2", "agent": "claude-code", "goal": "Eliminate intermittent connection timeouts under load", "current_state": "Race condition identified in pool acquire path. Fix drafted.", "next_action": "Write concurrent request tests, then open PR", "outcome_definition": "Zero timeout errors in load test at 500 rps" }
New session picks up GET /v1/tasks?project=api-v2&status=in_progress // Returns work items with full structured state + auto-generated event history. // Agent reads current_state and next_action — continues without re-investigation. // Artifacts (branch, PR, commits) are first-class attached objects, not buried in notes.
