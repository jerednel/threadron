Quickstart

Three steps to shared state

01

Get your API key

Sign up for the managed service, or spin up the Docker container and generate a key locally.

Managed # Sign up at tasksforagents.com # Your API key appears in the dashboard under Settings > API
Self-hosted docker compose up -d curl http://localhost:8080/v1/auth/setup
02

Add the skill to your agent

Drop the skill.md into your agent's configuration. It tells the agent how to check in, claim work, and update state on each session start.

skill.md # Threadron — Shared Execution State # Add to your agent's skill or system prompt file TASK_API=https://api.tasksforagents.com/v1 TASK_API_KEY=tfa_sk_... AGENT_ID=claude-code # On session start: # 1. GET $TASK_API/tasks?assignee=$AGENT_ID&status=in_progress # 2. Read current_state + next_action. Continue from there. # 3. If nothing in progress, GET unclaimed pending work and claim one.
03

Create your first work item

From the dashboard, your agent, or a simple curl. Use the structured fields — goal, current_state, outcome_definition — so agents have real context to work from.

curl curl -X POST https://api.tasksforagents.com/v1/tasks \ -H "Authorization: Bearer tfa_sk_..." \ -H "Content-Type: application/json" \ -d '{ "title": "Fix connection pool race condition", "project": "api-v2", "assignee": "claude-code", "goal": "Eliminate intermittent connection timeouts under load", "current_state": "Race condition identified in acquire path.", "next_action": "Write concurrent request tests, then open PR", "outcome_definition": "Zero timeout errors in load test at 500 rps" }' // Work item created. Claude Code picks it up on next session start.
Agent setup

Connect your agent

Two ways to connect — native MCP tools for Claude Code and OpenClaw, or REST via skill.md for any agent that can make HTTP requests.

MCP Server

Claude Code

Native tools, zero-config behavior. Claude Code discovers Threadron tools automatically via the MCP server.

Step 1 — Install the MCP server
bash cd your-project npm install threadron-mcp
Step 2 — Add .mcp.json to project root
.mcp.json { "mcpServers": { "threadron": { "command": "node", "args": ["node_modules/threadron-mcp/dist/index.js"], "env": { "TFA_API_URL": "https://api.tasksforagents.com/v1", "TFA_API_KEY": "tfa_sk_...", "TFA_AGENT_ID": "claude-code" } } } }
Step 3 — Add to CLAUDE.md
CLAUDE.md ## Threadron Use Threadron tools to track work across sessions: - Start each session with `threadron_checkin` - Before starting work, `threadron_claim` the item - Update `threadron_update_state` as you make progress - Record decisions with `threadron_add_context` - Attach outputs with `threadron_create_artifact` - When done or pausing, `threadron_release` the item
MCP Server

OpenClaw

Same MCP protocol, different agent. OpenClaw supports MCP servers — same config, different agent ID.

Step 1 — Install the MCP server
bash cd your-project npm install threadron-mcp
Step 2 — Add .mcp.json with your agent ID
.mcp.json { "mcpServers": { "threadron": { "command": "node", "args": ["node_modules/threadron-mcp/dist/index.js"], "env": { "TFA_API_URL": "https://api.tasksforagents.com/v1", "TFA_API_KEY": "tfa_sk_...", "TFA_AGENT_ID": "openclaw" } } } }
Step 3 — Verify tools are visible
OpenClaw # Tools should appear automatically once .mcp.json is present. # Look for threadron_checkin, threadron_claim, etc. # Instruct your agent to call threadron_checkin on session start.
REST API

Any Agent

Works with anything that can make HTTP requests. Copy skill.md into your agent's system prompt — it contains full API documentation.

Step 1 — Copy skill.md into your agent
bash # Copy skill.md from the repo into your agent's # system prompt, instruction file, or config. # It contains complete API documentation. curl -O https://raw.githubusercontent.com/threadron/threadron/main/skill.md
Step 2 — Set environment variables
env TFA_API_URL=https://api.tasksforagents.com/v1 TFA_API_KEY=tfa_sk_... TFA_AGENT_ID=hermes
Step 3 — Agent makes HTTP calls directly
REST # Session start — check what's in progress GET $TFA_API_URL/tasks?assignee=$TFA_AGENT_ID&status=in_progress # Claim before working POST $TFA_API_URL/tasks/{id}/claim # Update state as you go PATCH $TFA_API_URL/tasks/{id}
MCP tools reference
10 tools
Tool Purpose
threadron_checkin Session start — returns in-progress, pending, and blocked work
threadron_list_tasks List / filter work items by status, assignee, domain, search
threadron_get_task Full work item with goal, state, timeline, artifacts
threadron_create_task Create with structured fields (goal, current_state, outcome)
threadron_update_state Update current_state, next_action, blockers, confidence
threadron_add_context Add timeline entries: observation, decision, action_taken, etc.
threadron_create_artifact Attach branches, PRs, commits, files, docs
threadron_claim Claim before working — prevents collisions, auto-expires
threadron_release Release claim when done or pausing
threadron_list_domains List available domains
threadron_list_agents List registered agents and activity
