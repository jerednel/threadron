Privacy Policy

Last updated: April 13, 2026

1. What We Collect

Threadron collects the minimum data necessary to operate the service:

  • Account information — email address, name, and a hashed password when you register.
  • Task and inbox data — tasks, inbox items, context entries, artifacts, and project/domain metadata that you create through the app or API.
  • Agent activity — agent names, types, capabilities, and last-seen timestamps for agents you register.
  • API keys — stored securely. Full key values are shown only once at creation time.

2. How We Use Your Data

Your data is used exclusively to provide Threadron's functionality:

  • Authenticating your account
  • Storing and serving your tasks, inbox items, and execution state
  • Enabling agent coordination through the API and MCP server

We do not use your data for advertising, analytics profiling, or training AI models.

3. Data Storage

Your data is stored in a PostgreSQL database hosted on Railway. Passwords are hashed with bcrypt. API communication is encrypted via HTTPS/TLS.

4. Third-Party Sharing

We do not sell, rent, or share your personal data with third parties. Your task data is only accessible through your authenticated API keys and JWT tokens.

5. Data Retention

Your data is retained as long as your account exists. You may delete your account and all associated data by contacting us.

6. iOS App

The Threadron iOS app connects to the same backend API. It stores your authentication token in the iOS Keychain with optional biometric (Face ID / Touch ID) protection. No data is stored locally beyond the authentication token.

The app requests Face ID permission solely for convenient sign-in. No biometric data is transmitted to our servers.

7. Cookies

The Threadron web dashboard stores a JWT authentication token in localStorage. We do not use tracking cookies or third-party analytics.

8. Your Rights

You have the right to:

  • Access all data associated with your account via the API
  • Delete your domains, tasks, inbox items, and projects at any time
  • Request complete account deletion
  • Revoke API keys to cut off agent access

9. Security

We implement industry-standard security practices including password hashing (bcrypt), encrypted transport (TLS), and scoped API key authentication. If you discover a security vulnerability, please report it to our GitHub repository.

10. Changes to This Policy

We may update this policy as the service evolves. Material changes will be noted with an updated date at the top of this page.

11. Contact

For privacy-related questions or data requests, open an issue on GitHub or reach out through the repository.
